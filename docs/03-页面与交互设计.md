# 页面与交互设计 - 微信记账小程序

> 版本：v1.0
> 说明：本文档描述页面结构、导航流程、关键交互。

## 1. 整体导航结构

底部 Tab 栏（3 个 Tab + 中间凸起按钮）：

```
┌──────────────────────────────────────────┐
│                                          │
│              [页面内容]                    │
│                                          │
│                                          │
├──────────────────────────────────────────┤
│  📋 账单     ╭───╮     📊 统计    👤 我的 │
│             │ ＋ │                       │
│             ╰───╯                       │
└──────────────────────────────────────────┘
```

| Tab | 路径 | 功能 |
|-----|------|------|
| 账单 | pages/index/index | 首页（账单列表 + 月度概览） |
| ＋ | pages/record/record | 记一笔（凸起按钮，点击跳转） |
| 统计 | pages/stats/stats | 统计分析 |
| 我的 | pages/mine/mine | 个人中心 |

**关键交互**：
- 点击顶部账本名 → 弹出账本切换器（Action Sheet）

- 点击账单条目 → 跳转 P3 详情
- 长按账单条目 → 快捷弹窗（删除 / 复制再记一笔）
- 下拉刷新、上拉加载更多

│ │ 4   5   6                │   │  ← 自定义数字键盘

## 4. 关键交互流程

### 4.1 新用户首次使用
```
进入小程序
  ↓
[首屏 Loading]
  ↓
微信授权（getUserProfile，第一次需要点一下）
  ↓
[空状态首页] "还没记账，点 ＋ 开始吧"
  ↓
点击 ＋ → 跳转记一笔页
```

### 4.2 邀请加入共享账本
```
账本主在 P7 点击"邀请 ＋"
  ↓
弹出分享面板（系统右上角"…"分享）
  ↓
选择朋友 / 群发送
  ↓
对方点击小程序卡片
  ↓
小程序识别 invite_code 参数
  ↓
跳转 P9 邀请加入页
  ↓
点击"同意并加入" → 写入 ledger_members
  ↓
进入该账本首页
```

### 4.3 切换账本
```
P1 顶部点击账本名
  ↓
弹出 Action Sheet（账本列表）
  ↓
点击某账本 → 切换并刷新数据
  ↓
全局状态记住"当前账本"
```

## 5. 视觉风格建议

- **主色**：根据 Logo 决定，建议偏柔和（如 #4A90E2、#52C41A）
- **风格**：扁平化 + 大量留白
- **字体**：跟随系统默认（PingFang / 苹方）
- **圆角**：统一 8px 或 12px
- **暗黑模式**：v1 不做，v2 再考虑

## 6. 空状态与异常处理

| 场景 | 处理 |
|------|------|
| 首次进入无数据 | 插画 + "还没记账，点 ＋ 开始吧" |
| 网络异常 | Toast"网络不太给力，再试一下" + 重试按钮 |
| 加载中 | 顶部进度条 + 列表骨架屏 |
| 删除二次确认 | Modal"确认删除？删除后不可恢复" |
| 邀请码失效 | "邀请已过期，请联系账本所有者" |
