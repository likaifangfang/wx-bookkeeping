# 数据库设计 - 微信记账小程序

> 数据库：微信云开发数据库（基于 MongoDB 的文档型数据库）
> 版本：v1.0

## 1. 设计原则

- **用户隔离**：所有业务数据通过 `openid` 或 `ledger_id` 隔离
- **软删除**：关键数据用 `is_deleted` 标记，不物理删除
- **冗余可控**：少量必要冗余字段（如 `category_name`），换取查询性能
- **时间字段**：所有集合统一带 `created_at`、`updated_at`
- **权限控制**：通过云函数 + 数据库权限规则双层保障
- **显式 _openid**：云函数中 `add()` 不会自动注入 `_openid`，必须手动写入

## 2. 集合（表）总览

| 集合名 | 中文名 | 说明 |
|--------|--------|------|
| users | 用户表 | 微信用户基本信息 |
| ledgers | 账本表 | 个人 / 共享账本 |
| ledger_members | 账本成员表 | 账本与用户的关联，含角色 |
| categories | 分类表 | 每个账本独立的分类 |
| records | 账单记录表 | 核心数据，每条收支记录 |
| invitations | 邀请记录表 | 共享账本的邀请凭证 |

## 3. 详细设计

### 3.1 users（用户表）

```javascript
{
  _id: "auto",                    // 系统生成
  _openid: "oXXXXXXXXXXX",        // 微信 openid（云函数中需显式写入！）
  nickname: "小明",                // 昵称
  avatar_url: "https://...",      // 头像 URL
  default_ledger_id: "xxx",       // 默认账本 ID
  settings: {                     // 个人设置
    reminder_enabled: false,      // 是否开启记账提醒
    reminder_time: "21:00",       // 提醒时间
    privacy_mode: false           // 隐私模式（金额打码）
  },
  created_at: Date,
  updated_at: Date
}
```

**索引**：`_openid`（唯一）

### 3.2 ledgers（账本表）

```javascript
{
  _id: "auto",
  name: "我的账本",               // 账本名称
  icon: "wallet",                 // 图标 key
  color: "#FF6B6B",               // 主题色
  currency: "CNY",                // 币种（v1 仅 CNY）
  type: "personal",               // personal / shared
  owner_openid: "oXXX",           // 所有者 openid
  member_count: 1,                // 成员数（冗余字段，便于列表显示）
  is_deleted: false,
  created_at: Date,
  updated_at: Date
}
```

**索引**：
- `owner_openid`
- `is_deleted`

**说明**：
- 用户首次登录自动创建一个 `type: personal` 的默认账本
- 共享账本由用户主动创建，`type: shared`

### 3.3 ledger_members（账本成员表）

```javascript
{
  _id: "auto",
  ledger_id: "xxx",               // 关联账本 ID
  openid: "oXXX",                 // 成员 openid
  nickname: "小明",                // 冗余昵称（成员加入时快照）
  avatar_url: "https://...",      // 冗余头像
  role: "owner",                  // owner / editor / viewer
  joined_at: Date,
  is_active: true,                // 是否仍在账本中（false 表示已退出/被移除）
  created_at: Date,
  updated_at: Date
}
```

**索引**：
- 复合索引：`ledger_id + openid`（唯一）
- `openid`（查询用户所有账本）
- `ledger_id`（查询账本所有成员）

**说明**：
- 创建账本时自动写入一条 owner 记录
- 退出账本时 `is_active = false`，保留历史

### 3.4 categories（分类表）

```javascript
{
  _id: "auto",
  ledger_id: "xxx",               // 所属账本
  name: "餐饮",                   // 分类名
  icon: "food",                   // 图标 key
  color: "#FF9F43",               // 颜色
  type: "expense",                // expense / income
  sort: 1,                        // 排序权重
  is_default: true,               // 是否系统默认分类
  is_hidden: false,               // 是否隐藏（不删除避免数据断链）
  created_at: Date,
  updated_at: Date
}
```

**索引**：
- 复合索引：`ledger_id + type + sort`

**说明**：
- 创建账本时批量插入默认分类
- 用户隐藏分类后，新记账时不显示，但历史数据仍可查看

### 3.5 records（账单记录表）⭐ 核心

```javascript
{
  _id: "auto",
  ledger_id: "xxx",               // 所属账本
  openid: "oXXX",                 // 记录创建者
  type: "expense",                // expense / income
  amount: 35.50,                  // 金额（精确到分，单位：元）
  category_id: "xxx",             // 分类 ID
  category_name: "餐饮",          // 冗余分类名（便于列表展示）
  category_icon: "food",          // 冗余分类图标
  category_color: "#FF9F43",      // 冗余分类颜色
  note: "和同事午餐",              // 备注
  images: [                       // 图片（云存储 fileID）
    "cloud://xxx.png"
  ],
  record_date: Date,              // 账单日期（用户选择）
  creator_nickname: "小明",        // 冗余创建者昵称（共享账本展示用）
  creator_avatar: "https://...",  // 冗余创建者头像
  is_deleted: false,
  created_at: Date,
  updated_at: Date
}
```

**索引**：
- 复合索引：`ledger_id + record_date`（账本时间倒序查询）
- 复合索引：`ledger_id + category_id`（按分类统计）
- `openid`（个人维度查询）
- `is_deleted`

**关键设计决策**：

1. **金额用 Number 类型**：云开发 MongoDB 支持 Decimal128，但前端处理麻烦。第一版用 Number（双精度浮点），通过约束 `Math.round(amount * 100) / 100` 保留两位小数避免精度问题。
2. **冗余分类信息**：避免列表渲染时关联查询 categories 集合，提高性能
3. **冗余创建者信息**：共享账本展示"谁记的"时不用再查 users 表
4. **record_date 与 created_at 分开**：用户可以补记昨天的账，二者意义不同

### 3.6 invitations（邀请记录表）

```javascript
{
  _id: "auto",
  ledger_id: "xxx",               // 邀请进入的账本
  inviter_openid: "oXXX",         // 邀请人
  invite_code: "AB12CD",          // 邀请码（6 位随机）
  expires_at: Date,               // 过期时间（默认 7 天）
  used_count: 0,                  // 已使用次数
  max_uses: 10,                   // 最大使用次数
  is_active: true,
  created_at: Date
}
```

**索引**：`invite_code`（唯一）、`ledger_id`

**说明**：
- 分享小程序卡片时，路径中带 `invite_code`
- 对方进入后用 `invite_code` 校验并加入账本

## 4. 权限规则（数据库安全规则）

云开发数据库支持声明式权限，配置如下：

| 集合 | 权限规则 |
|------|----------|
| users | 仅创建者可读写 |
| ledgers | 仅所有者可写；成员可读（通过云函数校验） |
| ledger_members | 云函数读写，前端只读自己的记录 |
| categories | 通过云函数读写，前端按账本成员身份只读 |
| records | 通过云函数读写，前端按账本成员身份只读 |
| invitations | 仅创建者可写，凭邀请码可读 |

> **关键策略**：所有跨用户写操作（如共享账本写入）都走云函数，前端只读。这样能稳稳防止越权。

## 6. 典型查询示例

### 6.1 查询用户的所有账本
```javascript
// 1. 查 ledger_members 拿到 ledger_id 列表
// 2. 用 _id in 查 ledgers
db.collection('ledger_members')
  .where({ openid: '_openid_', is_active: true })
  .get()
  .then(res => {
    const ledgerIds = res.data.map(m => m.ledger_id)
    return db.collection('ledgers')
      .where({ _id: db.command.in(ledgerIds), is_deleted: false })
      .get()
  })
```

### 6.2 查询某账本本月账单
```javascript
db.collection('records')
  .where({
    ledger_id: 'xxx',
    record_date: db.command.gte(monthStart).and(db.command.lte(monthEnd)),
    is_deleted: false
  })
  .orderBy('record_date', 'desc')
  .limit(20)
  .get()
```

### 6.3 按分类统计本月支出（云函数中聚合）
```javascript
// 云函数中执行 aggregate
db.collection('records').aggregate()
  .match({
    ledger_id: 'xxx',
    type: 'expense',
    record_date: $.gte(monthStart),
    is_deleted: false
  })
  .group({
    _id: '$category_id',
    total: $.sum('$amount'),
    count: $.sum(1)
  })
  .sort({ total: -1 })
  .end()
```

## 7. 数据初始化

新用户登录时（云函数 `onUserLogin`）：

```
1. 检查 users 表是否有该用户
   - 没有 → 创建 user 记录
2. 检查是否有默认账本
   - 没有 → 创建 personal 账本，写入 ledger_members（owner）
3. 给该账本批量插入默认分类
4. 更新 users.default_ledger_id
5. 返回 { user, defaultLedger }
```

## 8. 容量评估（个人版免费额度）

按照云开发个人版免费额度（2GB 数据库 + 5万次/天读 + 3万次/天写）评估：

| 集合 | 单条大小 | 1万用户量 |
|------|---------|----------|
| users | ~500 B | 5 MB |
| ledgers | ~300 B | 3 MB（人均 1 账本） |
| ledger_members | ~400 B | 4 MB |
| categories | ~200 B | 32 MB（人均 16 个分类） |
| records | ~600 B | 1.2 GB（人均 200 条） |
| invitations | ~300 B | 1 MB |
| **合计** | | **~1.25 GB** |

**结论**：1 万用户内、人均 200 条记录的规模，免费额度完全 hold 住。
