# 技术架构 - 微信记账小程序

> 版本：v1.0

## 1. 技术栈总览

| 层 | 选型 | 说明 |
|----|------|------|
| 前端框架 | 原生微信小程序 | 无额外 UI 库依赖，纯手写样式 |
| 后端 | 微信云开发（CloudBase） | 免运维、零成本 |
| 数据库 | 云开发数据库（MongoDB-like） | 文档型 |
| 计算 | 云函数（Node.js） | 所有读写操作均通过云函数 |
| 鉴权 | 微信 openid | 自动获取，无需额外登录系统 |

## 2. 整体架构图

```
┌──────────────────────────────────────────────────────┐
│                  微信小程序前端                         │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐     │
│  │  页面 Pages │  │  Store 状态 │  │ 工具 Utils │     │
│  └────────────┘  └────────────┘  └────────────┘     │
│  ┌──────────────────────────────────────────────┐   │
│  │           Service 层（云函数调用封装）           │   │
│  └──────────────────────────────────────────────┘   │
└────────────────────────┬─────────────────────────────┘
                         │ wx.cloud.callFunction
                         ▼
              ┌──────────────────┐
              │   云函数 (19个)    │
              │  读写 + 校验 + 聚合 │
              └────────┬─────────┘
                       ▼
              ┌─────────────────┐
              │   云数据库        │
              │  6 个集合        │
              └─────────────────┘
```

**核心策略**：
- 所有数据操作（读 + 写）均通过云函数完成
- 云函数内校验用户身份和权限
- 前端通过 service 层统一调用云函数

## 4. 云函数设计

### 4.1 共享模块

所有云函数通过 `require('./_shared')` 引入共享工具：

```javascript
// _shared.js 提供：
const { ok, fail } = require('./_shared')  // 统一响应格式
const { DEFAULT_EXPENSE_CATEGORIES, DEFAULT_INCOME_CATEGORIES } = require('./_shared')  // 默认分类
```

部署方式：将 `cloudfunctions/_shared/utils.js` 复制为各云函数目录下的 `_shared.js`。

### 4.2 onUserLogin（用户登录初始化）

**触发**：小程序启动时调用

**关键逻辑**：
1. 通过 `cloud.getWXContext()` 获取 OPENID
2. 查 users 表（`where({ _openid: OPENID })`）
   - 不存在 → 创建用户记录（**显式写入 `_openid` 字段**）+ 创建默认共享账本 + 写入成员 + 批量写入默认分类
   - 存在 → 更新昵称头像
3. 查找有效的默认账本（三级 fallback）：
   - 优先用 `default_ledger_id`
   - 其次从 `ledger_members` 找
   - 再次从 `ledgers` 表按 `owner_openid` 找
4. 返回 `{ user, defaultLedger }`

**注意**：云函数中 `db.collection().add()` 不会自动注入 `_openid`，必须手动写入。

### 4.3 createRecord（创建账单）

**输入校验**：
- type 必须为 expense/income
- 备注 ≤200 字符
- 图片 ≤9 张
- 金额 ≤99999999

**逻辑**：校验成员权限 → 查分类信息 → 冗余写入分类名/图标/颜色 → 插入 records

### 4.4 joinLedger（加入共享账本）

**原子性保证**：使用条件更新 `where({ used_count: _.lt(max_uses) }).update()` 防止并发超额。更新失败则回滚成员记录。

### 4.5 deleteLedger（删除账本）

**批量处理**：云数据库 `.where().update()` 单次最多影响 20 条，使用 `while` 循环确保所有关联数据（members、records、invitations）全部处理完毕。

### 4.6 statsAggregate（统计聚合）

支持两种聚合模式：
- `monthOverview`：月度收支总览
- `aggregate`：按分类分组汇总

## 5. 前端关键模块

### 5.1 全局状态管理

极简发布订阅模式：

```javascript
// store/index.js
const state = {
  user: null,
  currentLedger: null,
  ledgers: [],
  categories: { expense: [], income: [] },
  _categoriesLedgerId: null,  // 分类缓存对应的账本 ID
  initError: null
}

// 页面订阅状态变化
const unsub = store.subscribe('currentLedger', (ledger) => { ... })
```

### 5.2 防重入机制

`app.js` 的 `initUser()` 使用 Promise 单例模式，防止并发调用多次触发 `onUserLogin`：

```javascript
initUser() {
  if (this._initPromise) return this._initPromise
  this._initPromise = this._doInitUser().finally(() => {
    this._initPromise = null
  })
  return this._initPromise
}
```

### 5.3 分类缓存

record 页面优先从 store 读取分类数据，同一账本不重复请求：

```javascript
const cached = store.getState('categories')
const cachedLedgerId = store.getState('_categoriesLedgerId')
if (cached && cachedLedgerId === ledger_id) { /* 使用缓存 */ }
```

### 5.4 智能刷新

首页 `onShow` 使用 `_needRefresh` 标记，仅在从记账/详情页返回时刷新，避免无谓请求。

## 6. 安全措施

| 措施 | 说明 |
|------|------|
| 身份校验 | 所有云函数校验 OPENID + 成员身份 |
| 输入校验 | 名称/备注/金额有长度和范围限制 |
| 权限控制 | owner/editor/viewer 三级角色 |
| 管理员保护 | migrateIcons 等管理工具有 OPENID 白名单 |
| 原子操作 | joinLedger 的邀请码消耗使用条件更新防竞态 |
| 批量安全 | deleteLedger 循环处理防止遗漏 |
| 查询限制 | 所有 `.get()` 显式指定 `.limit(100)` |

## 7. 性能优化

- **分类缓存**：store 级别缓存，切换账本时失效
- **按需刷新**：`_needRefresh` 标记避免 onShow 重复请求
- **分页加载**：账单列表每页 20 条，触底加载更多
- **预创建邀请码**：进入账本设置页时后台创建，点击分享时即可使用

## 8. 开发与部署

### 开发流程

```
1. 微信开发者工具打开项目
2. 修改前端代码 → 自动编译 → 模拟器预览
3. 修改云函数 → 右键"上传并部署：云端安装依赖"
4. 真机预览 → "预览"按钮 → 手机扫码
```

### 共享模块同步

修改 `_shared/utils.js` 后，在项目根目录执行：

```bash
cd cloudfunctions
for d in $(ls -d */ | grep -v _shared | grep -v node_modules); do
  cp _shared/utils.js "${d}_shared.js"
done
```

### 发布流程

```
1. 开发者工具 → "上传" → 填写版本号
2. mp.weixin.qq.com → 版本管理 → 提交审核
3. 审核通过 → 发布
```

## 9. 编码规范

- **文件命名**：kebab-case
- **变量命名**：camelCase
- **常量**：UPPER_SNAKE_CASE
- **数据库字段**：snake_case
- **云函数响应**：统一 `{ code: 0, message: 'ok', data }` / `{ code: -1, message: '...', data: null }`
